# Результаты сравнения алгоритмов рекомендаций

## Методология оценки

Оценка проводится методом трёхкратной перекрёстной проверки (сиды: 2026, 2027, 2028), k = 10 рекомендаций, разделение train/test 80/20 для каждого пользователя.

**Набор данных:**
- 120 реальных товаров из Kaggle (Fashion Product Images Small)
- 500 пользователей со структурированными профилями предпочтений (15% cold-start, 60% сфокусированных, 25% разносторонних)
- ~6 500 взаимодействий (предпочтения + 5% межкатегориального шума)
- 8 подкатегорий: Topwear, Bottomwear, Shoes, Sandal, Bags, Watches, Jewellery, Fragrance

**Формула итоговой точности:**
```
Accuracy = Hit Rate * 0.50 + NDCG * 0.30 + Precision * 0.20
```

## Итоговые результаты (среднее по 3 сидам, k = 10)

| Ранг | Алгоритм | Accuracy | HR@10 | Precision@10 | NDCG@10 | MRR |
|------|----------|----------|-------|--------------|---------|-----|
| 1 | **Hybrid** | **85.79%** | 1.000 | 0.8367 | 0.6351 | 0.6145 |

| 4 | Item-Based CF | 85.39% | 1.000 | 0.8273 | 0.6280 | 0.6035 |

Полные метрики доступны на панели мониторинга: http://localhost:8000/analytics/

## Ключевые выводы

**Гибридная система** показывает наилучший результат по итоговой метрике Accuracy (85.79%), а также лучшие значения Hit Rate@10 (1.000) и NDCG@10 (0.6351) среди всех пяти алгоритмов.

**Гибридная система выбрана в качестве производственной модели**, потому что:
- объединяет сигналы всех четырёх базовых алгоритмов через взвешенный ансамбль (rank-decay scoring) с усилением по консенсусу
- устойчива к разреженным данным и проблеме холодного старта благодаря компоненте Content-Based
- комбинирует преимущества нескольких подходов, не завязываясь на один источник сигнала
- менее подвержена переобучению под конкретный паттерн данных

**SVD** показывает наилучшую отдельную метрику Precision@10 (0.8387), однако имеет немного более низкий Hit Rate@10 (0.996) — латентные факторы иногда упускают однозначные категориальные предпочтения.

**Content-Based фильтрация** показывает наименьшие значения среди пяти алгоритмов (Accuracy = 85.04%), что ожидаемо: TF-IDF-признаки строятся на основе сравнительно коротких текстовых описаний товаров из CSV-файла и не используют информацию о поведении других пользователей.

**Разница между алгоритмами невелика (85.04% – 85.79%)**, что характерно для рекомендательных систем: даже небольшое, но устойчивое (повторяющееся на всех трёх сидах) преимущество гибридного подхода является значимым результатом.

## Веса гибридной системы

| Алгоритм | Вес |
|----------|-----|
| Item-Based CF | 40% |
| User-Based CF | 35% |
| SVD | 15% |
| Content-Based | 10% |

Веса заданы в `RecommendationService.HYBRID_WEIGHTS` и используются в формуле rank-decay scoring:

```
score(item) = Σ HYBRID_WEIGHTS[algo] × (n - rank) / n
```

---

## Метрики оценки

| Метрика | Описание |
|---------|---------|
| Precision@10 | Доля рекомендованных товаров, относящихся к предпочтительной категории пользователя |
| Hit Rate@10 | Доля пользователей, получивших хотя бы одну релевантную рекомендацию |
| NDCG@10 | Нормализованный дисконтированный накопленный выигрыш — качество ранжирования |
| MRR | Среднее обратное ранга — как быстро появляется первый релевантный результат |
| Recall@10 | Доля релевантных товаров, попавших в топ-10 рекомендаций |
| F1 | Гармоническое среднее Precision и Recall |
| Accuracy | Итоговая метрика: HR×0.50 + NDCG×0.30 + Precision×0.20 |

---

## Ограничения

Взаимодействия пользователей сгенерированы синтетически на основе реального каталога товаров Kaggle для обеспечения контролируемой оценки. Полученные метрики используются для **относительного сравнения** алгоритмов, а не как абсолютные показатели качества в продакшен-среде.

## Воспроизведение результатов

```bash
# Сгенерировать бенчмарк-данные (требуется data/fashion-product-images-small/styles.csv)
python manage.py generate_benchmark_data

python manage.py compare_algorithms

python manage.py runserver
```
