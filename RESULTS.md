# Результаты сравнения алгоритмов рекомендаций

## Методология оценки

Оценка проводится методом трёхкратной перекрёстной проверки (сиды: 2026, 2027, 2028), k = 10 рекомендаций.

**Набор данных:**
- 120 реальных товаров из Kaggle (Fashion Product Images Small)
- 500 пользователей со структурированными профилями предпочтений
- 6 000+ взаимодействий (предпочтения + 5% шума)
- 8 подкатегорий: Topwear, Bottomwear, Shoes, Sandal, Bags, Watches, Jewellery, Fragrance

**Формула итоговой точности:**
```
Accuracy = Hit Rate * 0.50 + NDCG * 0.30 + Precision * 0.20
```

## Ключевые выводы

**Item-Based CF** показывает наилучшую точность (Precision@10 = 2.63%, NDCG@10 = 8.59%), поскольку в наборе данных пользователи имеют устойчивые категориальные предпочтения, хорошо улавливаемые матрицей сходства товаров.

**Гибридная система** выбрана в качестве производственной модели несмотря на незначительно более низкие абсолютные показатели, потому что:
- устойчива к разреженным данным
- комбинирует преимущества нескольких подходов
- в итоговый список попадают только товары, получившие поддержку от нескольких алгоритмов
- менее подвержена переобучению под конкретный паттерн данных

**Низкие абсолютные значения ожидаемы** при работе с реальными данными: 6 000 взаимодействий для 120 товаров и 500 пользователей — разреженная матрица (заполненность < 10%).

**Content-Based фильтрация** показывает наихудшие результаты, поскольку возможности TF-IDF ограничены при коротких текстовых описаниях товаров из CSV-файла.

## Метрики оценки

| Метрика | Описание |
|---------|---------|
| Precision@10 | Доля рекомендованных товаров, относящихся к предпочтительной категории пользователя |
| Hit Rate@10 | Доля пользователей, получивших хотя бы одну релевантную рекомендацию |
| NDCG@10 | Нормализованный дисконтированный накопленный выигрыш — качество ранжирования |
| MRR | Среднее обратное ранга — как быстро появляется первый релевантный результат |
| Recall@10 | Доля релевантных товаров, попавших в топ-10 рекомендаций |
| F1 | Гармоническое среднее Precision и Recall |
| Diversity | Разнообразие рекомендаций (попарное косинусное несходство) |
| Coverage | Доля каталога, охваченная рекомендациями |
